PABLO: Learning Personalized Agent Orchestration with Factorized Bandits

A lightweight, user-specific policy model that learns how a frozen black-box agent should use memory, acquire information, and present a response.

PABLO updates online from each user's partial feedback while deterministic constraints control mandatory confirmation and safety behavior.

Overview

Personalizing the orchestration layer

PABLO personalizes the orchestration layer: which memory to use, how to acquire missing information, and how to present the response.

Modern personal agents make intermediate execution choices before producing natural language. PABLO casts the learnable choices as a contextual bandit: each request creates a context, a deterministic constraint layer removes infeasible choices, and the policy observes feedback only for the selected action.

The learnable action is factorized as a = (m, i, p), where m selects memory, i selects information acquisition, and p selects response presentation. Mandatory confirmation and safety rules are deliberately outside this action space. To our knowledge, PABLO is the first framework to learn a lightweight user-specific policy online for controlling multiple behavioral decisions of a frozen, black-box LLM agent.

Feedback model

E[y | c, a] = b(c, a) + φ(c, a)Tθ − λ · cost(c, a)

b(c,a)

Default system behavior and rule-based orchestration utility.

φ(c,a)Tθ

User-specific residual preference learned online.

λ cost(c,a)

Explicit penalty for latency, information acquisition, or verbosity.

Method

Constraint-aware factorized learning

PABLO is an orchestration plugin for a larger assistant runtime. The host agent still executes retrieval, information acquisition, clarification, and response generation.

1

Deterministic constraint layer

Applies memory permissions, tool availability, mandatory confirmations, and safety rules before bandit selection. These requirements are never learned from reward.

2

Factorized Thompson sampling

Scores admissible actions using default utility, sampled user residual preference, and explicit cost while sharing signal across memory, acquisition, and presentation components.

3

Per-user residual update

Updates a compact per-user posterior after each feedback event. Preference persistence is evaluated separately; the current evidence does not establish a promotion benefit.

Orthogonal action schema

PABLO separates learnable behavioral choices from non-negotiable constraints, so component effects remain interpretable.

Memory selection
no memory, recent memory, semantic memory, preference memory, profile summary
Information acquisition
no acquisition, web search, file search, code execution, preference checker, clarification
Response presentation
direct, concise, detailed, step-by-step
Deterministic rules
mandatory confirmation, safety gates, permissions, and feasibility checks remain outside the bandit

Algorithm

Online personalization loop

The loop residualizes feedback against default behavior and explicit cost, then updates a Bayesian posterior over user-specific orchestration preferences.

Evidence boundary

Reward and success do not rank policies consistently: Global LinTS has higher reported reward than PABLO full on both tasks, while its success is near zero. The current artifact therefore does not establish that the reward is a faithful proxy for task success.

The full model does not establish a promotion benefit: its embodied success is 7.8%, compared with 13.2% for the no-promotion ablation. Promotion, calibrated uncertainty, and safety violations need separate evaluation.

Limitations

Limitations and next steps

Metric alignment

Reward, regret, and task success can produce different rankings in the current artifact. Future experiments should pre-register a primary task-success metric and diagnose reward calibration against it.

Uncertainty and replication

The released summaries have 20 simulated users and 30 rounds each, but no independent-seed aggregation, standard errors, or confidence intervals. These are required before comparative claims.

Safety and persistence

Mandatory confirmations are now deterministic constraints rather than learnable actions. Preference persistence and safety violations need dedicated end-to-end evaluation before they become outcome claims.
